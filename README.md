В данном репозитории есть: 
1) код на C++, написанный в Visual Studio Code, для уникальной платы в основе которой находится микроконтроллер ATmega128. (в основном файле "TU_SB_03_1.ino" - кодировка OEM866, в сотальных - стандартная UTF-8. Для удобства приложу данный файл в текстовом формате.)
2) Универсальное приложение (исходники + "собранное" приложение), написанное на Qt.
3) XML-файл - конфиг-файл, подгружаемый в MK_Interface (название приложения)

Описание и принцип работы:
Плата, в основе которой находится микроконтроллер, содержит много элементов, входов, выходов, интерфейсы передачи данных. После ее распайки и сборки необходимо проверить ее работоспособность. Вручную прозванивать это очень долго, плюс ко всему, была необходимость в "стресс-тесте" данной платы. Под стресс-тестом подразумевается полная НЕПРЕРЫВНАЯ проверка функционала в течении 48-ми часов. Для автоматизиции был написан код под микроконтроллер с использованием собственной написанной вручную библиотеки под МСР2515 (ее писал не я, библиотека не сохранилась в моих архивах), а для взаимодействия и контроля платы было написано УНИВЕРСАЛЬНОЕ приложение на Qt.

Почему MK_Interface универсальный? 
В работе было очень много плат, с которыми можно взаимодействовать через одно и то же приложение. Для этого был придуман проприетарный протокол общения. 

ТАКЖЕ!!!
Функционал каждой платы/блока менялся, но проверка была у всех похожа. Для этого в MK_Interface было придумано использовать конфиг-файл (XML-файл), загружая который, приложение немного меняет внешний вид и функционал. Это упростило работу, так как полностью отпала надобность написания нового приложения, а всего лишь нужно было написать короткий XML-файл для нового изделия. Функция парсинга данных из XML-файла есть в MK_Interface.

Данное приложение устанавливалось на любой ноутбук/ПК с windows. В конкретном случае: программу "ставили" на ноутубук, подключались к плате с помощью преобразователя интерфейсов EL205-1 (преобразователь USB-CAN). Данный преобразователь эмулировал COM-порт.
